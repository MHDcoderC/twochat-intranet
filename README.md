# Two Chat

پیام‌رسان دو نفره سبک برای استفاده در شبکه داخلی (LAN/Intranet) و گفت‌وگوهای شخصی.

## مناسب چه سناریویی است

- گفت‌وگوی دو نفره در شبکه داخلی سازمان یا خانه
- اجرای روی سرور داخلی بدون نیاز به سرویس خارجی
- ارسال پیام متنی، تصویر و صدا با رابط موبایل‌فرست

## امکانات

- پیام متنی، تصویر و صدا
- ارتباط بلادرنگ با `SSE` (پیام جدید، حضور، خوانده شدن، ویرایش، ری‌اکشن، تایپ)
- ذخیره‌سازی محلی پیام‌ها در `data/messages.json`
- آپلود فایل‌ها در `uploads/` (آدرس با نام شبه‌تصادفی؛ درخواست‌ها با `?username=` هویت می‌گیرند)
- فشرده‌سازی خودکار تصویر با `sharp`
- ایموجی‌پیکر و استیکر داخلی بدون CDN خارجی
- احراز هویت برای دو کاربر مجاز (`ALLOWED_USERS`)؛ در کلاینت معمولا از پارامتر `?username=` روی API استفاده می‌شود تا در چند تب یک مرورگر کوکی سشن با هم قاطی نشود.

### امکانات گفت‌وگو (جزئیات رفتار)

- **ریپلای:** روی دکمه ریپلای کنار هر پیام بزنید؛ بالای باکس ارسال نوار «ریپلای» ظاهر می‌شود و پیام جدید به همان پیام پیوند داده می‌شود. در ذخیره و نمایش، پیش‌نمایش ریپلای در حباب پیام دیده می‌شود.
- **ویرایش پیام:** فقط برای پیام‌های **متنی** و فقط **از طرف فرستنده**. آیکون ویرایش کنار پیام خروجی؛ با کلیک متن در باکس قرار می‌گیرد و با ارسال، همان شناسه پیام از طریق `PATCH /api/messages/:id` به‌روزرسانی می‌شود و برای طرف مقابل با رویداد `message:update` ریل‌تایم می‌آید. برچسب «ویرایش‌شده» روی پیام نمایش داده می‌شود.
- **خوانده‌شدن (تیک‌ها):** هر پیام فیلد `readBy` دارد. گیرنده با باز بودن چت، پیام‌های ورودی را «خوانده» ثبت می‌کند (`POST /api/messages/read`). فرستنده تیک ارسال/خوانده‌شدن را روی پیام‌های خود می‌بیند؛ آپدیت با SSE `message:read` و همگام‌سازی دوره‌ای اختیاری است.
- **ری‌اکشن:** دکمه ری‌اکشن کنار پیام؛ پنجره کوچک با ایموجی‌های 💕 ❤️ 😍 👌. انتخاب همان ایموجی را به پیام می‌چسباند یا اگر دوباره بزنید برمی‌دارد (`POST /api/messages/:id/reactions`). شمارنده روی پیام و broadcast با `message:update`.
- **در حال تایپ:** هنگام تایپ در باکس پیام، درخواست سبک به `POST /api/typing` با `typing: true/false`؛ طرف مقابل زیر عنوان گفت‌وگو متن «در حال تایپ…» را از رویداد `typing:update` می‌بیند. با توقف تایپ یا ارسال پیام، تایپ خاموش می‌شود.
- **آنلاین / آفلاین:** `POST /api/presence` و رویداد `presence:update`؛ هنگام اتصال به `/api/events` یک snapshot از وضعیت فعلی هم ارسال می‌شود تا دیر وارد شونده بلافاصله وضعیت را ببیند. در `requireAuth` اولویت با `?username=` است تا دو کاربر در یک مرورگر اشتباهی به هویت دیگری نخورند.

## اجرای سریع

```bash
npm install
cp .env.example .env
npm start
```

سرویس به طور پیش‌فرض روی `http://localhost:3000` اجرا می‌شود.

## تنظیمات محیطی

فایل `.env`:

```env
PORT=3000
ALLOWED_USERS=user1,user2
SESSION_SECRET=replace-with-a-long-random-secret
MAX_MESSAGES=1000
NODE_ENV=production
TRUST_PROXY=false
```

- `ALLOWED_USERS`: دقیقا ۲ نام کاربری، با کاما جدا شود.
- `SESSION_SECRET`: مقدار طولانی و تصادفی (حداقل 32 کاراکتر).
- `TRUST_PROXY=true`: فقط زمانی فعال شود که پشت reverse proxy هستید.

نمونه استقرار عمومی (بدون اطلاعات شخصی):

```env
PORT=YOUR_INTERNAL_PORT
ALLOWED_USERS=user_a,user_b
NODE_ENV=production
TRUST_PROXY=true
```

## راهنمای استقرار روی سرور داخلی

1. **Node.js LTS** نصب کنید (ترجیحا نسخه 20 یا بالاتر).
2. پروژه را روی سرور داخلی clone کنید.
3. فایل `.env` را با مقادیر واقعی تنظیم کنید.
4. در شبکه داخلی پورت سرویس را فقط برای IPهای مجاز باز کنید.
5. برای اجرا در پس‌زمینه از `pm2` یا سرویس سیستم‌عامل استفاده کنید.

نمونه با `pm2`:

```bash
npm i -g pm2
pm2 start server.js --name two-chat
pm2 save
```
