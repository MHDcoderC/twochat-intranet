**پرامپت طراحی ضد-کلیشه‌ی AI**

طراحی [نوع پروژه] رو با رویکرد زیر انجام بده. هدف اینه که نتیجه نهایی حس "تصمیم طراحی آگاهانه" بده، نه "خروجی پیش‌فرض یه ابزار AI". قبل از شروع، یک هویت بصری (Design Identity) مشخص برای این پروژه تعریف کن و در تمام تصمیمات بعدی بهش پایبند باش.

**۱. هویت و شخصیت بصری**
قبل از کدنویسی، سه کلمه که شخصیت برند رو توصیف می‌کنن انتخاب کن (مثلاً: جسور/مینیمال/گرم، یا رسمی/دقیق/سرد). تمام تصمیمات رنگ، فونت و چیدمان باید این سه کلمه رو منعکس کنن. یک رقیب یا مرجع بصری مشخص نکن، چون این باعث میشه به سمت میانگین safe بری.

**۲. تایپوگرافی**
از Inter، Roboto، Poppins، یا فونت‌های سیستمی پیش‌فرض استفاده نکن.
یک فونت اصلی برای عناوین و یک فونت متفاوت (یا همون با وزن‌های متفاوت) برای متن انتخاب کن که با هم کنتراست شخصیتی دارن.
از مقیاس تایپوگرافی غیرخطی استفاده کن (نه فقط ۱۶، ۲۴، ۳۲؛ یه scale هدفمند مثل ۱.۲ یا ۱.۳۳ ratio بساز).
حداقل یک عنوان بزرگ رو به‌عنوان عنصر گرافیکی استفاده کن، نه فقط متن.
letter-spacing و line-height رو دستی تنظیم کن، نه مقادیر پیش‌فرض فریم‌ورک.

**۳. رنگ**
گرادینت بنفش-آبی یا صورتی-بنفش به‌هیچ‌وجه استفاده نشه.
یک پالت کامل بساز: یک رنگ پایه، یک رنگ ثانویه، یک رنگ تاکیدی (accent) غیرمنتظره، و طیف خاکستری سفارشی (نه gray-100 تا gray-900 پیش‌فرض Tailwind بدون تغییر).
نسبت کنتراست رنگی رو برای دسترسی‌پذیری (WCAG AA حداقل) رعایت کن.
حالت دارک‌مود رو اگه لازمه، به‌عنوان یه پالت مستقل طراحی کن نه فقط معکوس کردن رنگ‌ها.

**۴. سیستم چیدمان و فاصله‌گذاری**
از فرمول "هیرو وسط‌چین + سه کارت + تستیمونیال + CTA" پیروی نکن.
یک grid system با ستون‌های نامتقارن (مثلاً 7/5 یا 8/4 به‌جای 6/6) تعریف کن.
یک واحد فاصله‌گذاری پایه (spacing unit) مشخص کن و کل صفحه رو بر اساس همون مضرب پیش ببر، نه فاصله‌های دلبخواهی.
از فضای سفید به‌عنوان ابزار هدایت چشم استفاده کن، نه فقط padding یکسان همه‌جا.
حداقل یک سکشن رو با چیدمان غیرمتعارف بساز (مثلاً متن و تصویر همپوشان، یا یه عنصر که از grid بیرون می‌زنه).

**۵. کامپوننت‌ها و المان‌های UI**
کارت‌های shadow نرم پیش‌فرض و border-radius یکسان رو حذف کن؛ یا کاملاً flat با border مشخص برو، یا یه سبک امضادار بساز (برش گوشه، لایه‌بندی، سایه سخت/hard shadow).
دکمه‌ها باید حالت‌های hover، active، disabled و focus مجزا و سفارشی داشته باشن، نه فقط opacity یا darken پیش‌فرض.
فرم‌ها و input ها رو با توجه به برند طراحی کن، نه استایل پیش‌فرض مرورگر یا کتابخونه.
از کامپوننت‌های shadcn/ui یا Material بدون تغییر ظاهری استفاده نکن؛ حداقل رنگ، radius، فونت و سایه‌شون رو سفارشی کن.

**۶. آیکون و تصویر**
آیکون‌های Lucide/Heroicons رو بدون تغییر استفاده نکن؛ stroke-width، اندازه و رنگشون رو سفارشی کن یا یه ست آیکون متفاوت انتخاب کن.
به‌جای آیکون، در جاهایی که ممکنه از تصویرسازی، عکاسی، یا گرافیک سفارشی استفاده کن.
از تصاویر استوک ژنریک (مثل عکس‌های افراد لبخندزن جلوی لپ‌تاپ) پرهیز کن.

**۷. عناصر تمایزبخش (Signature Elements)**
حداقل یک عنصر بصری منحصربه‌فرد طراحی کن که فقط مال این پروژه باشه: پترن پس‌زمینه سفارشی، شکل هندسی تکرارشونده، خط جداکننده غیرمعمول، یا یه نوع تعامل خاص.
این عنصر باید در چند جای سایت تکرار بشه تا حس یکپارچگی برند ایجاد کنه.

**۸. حرکت و تعامل**
از انیمیشن‌های fade-in/slide-up ژنریک با easing پیش‌فرض پرهیز کن.
برای انیمیشن‌ها از easing سفارشی (cubic-bezier) استفاده کن که حس مکانیکی یکسان نده.
تعامل‌ها (hover، scroll، click) باید بازخورد بصری متفاوت و هدفمند داشته باشن، نه فقط opacity یا scale ساده.
از انیمیشن صرفه‌جویی کن؛ هر حرکت باید دلیل داشته باشه، نه تزئین صرف.

**۹. لحن نوشتاری و میکروکپی**
از جملات عمومی و کلیشه‌ای مثل "Empower your business" یا "Unlock your potential" پرهیز کن.
متن دکمه‌ها و CTA ها باید مشخص و اختصاصی باشن (نه فقط "Get Started" یا "Learn More" بدون context).
لحن نوشتار باید با هویت برند تعریف‌شده در بخش اول هماهنگ باشه.

**۱۰. واکنش‌گرایی و حالت‌های مختلف**
طراحی باید در موبایل، تبلت و دسکتاپ هر کدوم چیدمان متناسب خودشون رو داشته باشن، نه فقط فشرده‌سازی همون چیدمان دسکتاپ.
حالت‌های loading، empty state، error state رو هم با همون هویت بصری طراحی کن، نه پیش‌فرض فریم‌ورک.

**۱۱. چک‌لیست نهایی ضد-کلیشه**
قبل از تحویل نهایی، این موارد رو بررسی کن:
- آیا از فونت Inter/Roboto/Poppins استفاده شده؟ (باید نه باشه)
- آیا گرادینت بنفش-آبی وجود داره؟ (باید نه باشه)
- آیا چیدمان هیرو وسط‌چین کلاسیک هست؟ (باید نه باشه)
- آیا آیکون‌ها بدون تغییر از کتابخونه پیش‌فرض گرفته شدن؟ (باید نه باشه)
- آیا حداقل یک عنصر بصری امضادار وجود داره؟ (باید بله باشه)
- آیا نتیجه نهایی با یه نگاه قابل تشخیص از قالب‌های SaaS رایجه؟ (باید نه باشه)
